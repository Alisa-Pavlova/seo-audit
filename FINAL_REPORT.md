## 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### 1. **Страница: `/all-coins-list` — КРИТИЧЕСКОЕ СОСТОЯНИЕ**

**Score: 40/100** (самый низкий рейтинг)

| Метрика | Значение | Норма | Статус |
|---------|----------|-------|--------|
| LCP | 23,340ms | < 2,500ms | 🔴 **+830% превышено** |
| TBT | 868ms | < 200ms | 🔴 **+334% превышено** |
| CLS | 0.068 | < 0.1 | ✅ OK |
| FCP | 3,085ms | < 1,800ms | ⚠️ Медленно |

**Проблемы:**
- Очень медленная загрузка контента (23.3 сек!)
- Огромное время блокировки потока (868ms)
- Это вероятно **страница со списком 1000+ монет** с тяжелой DOM-структурой

**Рекомендации:**
1. Внедрить виртуализацию списка (virtualscroll)
2. Ленивая загрузка изображений (lazy loading)
3. Разбить на страницы или бесконечный скролл с load-on-demand
4. Оптимизировать JavaScript бандл

```typescript
// ДО: загружать все 1000+ элементов в DOM
<div>
  {coins.map(coin => <CoinRow coin={coin} />)}
</div>

// ПОСЛЕ: виртуализировать видимые элементы
import { FixedSizeList } from 'react-window';

<FixedSizeList
  height={600}
  itemCount={coins.length}
  itemSize={50}
>
  {({ index, style }) => (
    <div style={style}>
      <CoinRow coin={coins[index]} />
    </div>
  )}
</FixedSizeList>
```

### 2. Ленивая загрузка изображений
```typescript
// ДО: загружать все изображения сразу
<img src={url} />

// ПОСЛЕ: загружать при видимости
<img loading="lazy" src={url} />
// или с Intersection Observer для большего контроля
```

### 3. Кэширование API результатов
```typescript
// Использовать SWR или React Query для кэширования

  revalidateOnFocus: false,

```
